v-model

v-model is what makes your inputs work. It allows us to transfer different types of input (text, checkboxes, select options, etc) into the vue app. To quote masteringjs.io ,
'Two way data binding is a powerful pattern for building JavaScript forms with Vue. For example, suppose you have an input element and a JavaScript variable value. Two way data binding means:
1. When the user types in the input, value gets updated to match the value in input.
2. When you update value, the input element's content updates to match value.'
Official v-model Docs
Two Way Binding Explained by masteringjs.io
Learn Vue.co

v-bind

v-bind will help vue communicate with your html; it doesn't add new functionality. v-bind is used to bind an html attribute to something created through javascript. A good rule of thumb is to ask yourself if the vue app needs to be able to read the attribute; if it does, you probably need v-bind. Some examples of html attributes are value, href, id, style, src, alt and key. See 'Select and Options(Dynamic)' for a detailed example. The shorthand for v-bind turns v-bind:value='color' into :value='color'. Click here for a complete list of html attributes

v-for

v-for runs a for loop using information from vue. It works just like any other for loop, we just add v-for so that the html page can read it from vue. As with any for loop, you may run into issues with the keys attribute, and telerik has an article that dives into into it.

v-if, v-else-if, v-else

v-if statements don't work like the standard if statements from JavaScript and Python. v-if statements are one of two ways we can conditionally render a block of code; they're attached to the parent element, and they will either render or not render the block of code. We're not writing our own conditions, just checking if the vue data variable is truthy or falsey. A variable is considered falsey if it is undefined, NaN (not a number), 0, or blank (such as a blank string ""); if the variable equals anything else, it's considered truthy. This will help us control when things are rendered, and can help us style our output. v-if and v-show are extremely similar, and the differences will be explained in the v-show section.
Official v-if Statement Docs
v-if and v-show

v-show

v-show is the other way we can conditionally render code. Like v-if, v-show is also checking if a data variable is truthy or falsy. They work nearly identically, and you'll only ever see a difference in performance. learnvue explains it as such,
"Just like all choices in development, itâ€™s important to carefully consider when to use v-if and when to use v-show. As a general rule for performance, v-if has higher toggle costs (whenever the conditional changes) and v-show has higher initial render costs. So if you need to toggle something frequently, use v-show. If the conditional does not change that frequently during the runtime, use v-if. Another thing to consider is that using v-if allows us to use other logic blocks in combination with it. Specifically, we can use v-else and v-else-if blocks to really build advanced logic into our app.""
Official v-show Docs
learnvue (v-if and v-show)

v-on

v-on is used for event handling. Remember the onClick event from vanilla javascript? This is the same thing, only in vue. v-on listens for any event you set it to, such as submit and click. Then you pass in the method you want performed when that event happens. That's it!
Official v-on Docs

shorthand

Vue provides shorter ways to use v-bind and v-on. Shorthand turns v-bind:value='color' into :value='color'. v-on:click='calculate' into @click='calculate'. It's the same thing, just a little bit shorter.

Setting Up Vue.js

Now we can go to our javascript file, and create a new Vue app (you can name it script.js if you're unsure). This is Vue in its most basic form, before we add functionality to it.



After The Setup

To make Vue and the html work together, we'll have three stations that we can focus on initially. There will be more later on, but this is all we need to get started. These stations are the html page, the data section of Vue, and the methods section of Vue. You don't have to start at any particular station, and you don't have to finish one to start another.


HTML Station

In our html station, we're building our visual display. We'll want to think about our input types (text, number, email, radio, select, checkbox, etc.), outputs, forms, buttons, and how we'll ultimately want them to react to user input. Look through the 'Vue HTML' section to see the most common Vue-specific html elements.



Data Station

We can think of the data station as real-time storage locker that we can access from anywhere in our html and vue pages. The data station is an object full of key/value pairs that will keep track of the state of all of our data. The key will be a data variable, and just like any other variable in programming, we'll name it based on what it's keeping track of. The value will establish both the data type, and the current state of our key; it can be left blank, or a value can be hard-coded into it. Here, we'll want to think about what data types we need for our input/output (arrays, numbers, strings, objects, etc.), as well as the initial value.



Methods Station

Here at the methods station, we can start writing functions to use in our app. None of these functions will run until they're called, and we can do that in a few different ways. We can call one function inside another right here in the methods, or use v-on to trigger the function after an event (such as a click). In the advanced topics section, we'll look at Vue's lifecycle hooks, which can also call a function from methods. We'll want to think about how data needs to be able to move, and how we'll add that functionality to it. We can use any variable from the data station as long as we add this. to it first.

Standard Checkboxes

Checkboxes are boolean values (checked or unchecked, true or false) that can bound to data variables in vue just like text and radio buttons. In Vue, the checkMePlease variable is empty (which makes it falsey), so the checkbox will be unchecked. uncheckMePlease has a value other than a blank string (which makes it truthy), so the checkbox will be checked. If two checkboxes are bound to the same data variable using v-model, they will both be checked/unchecked simultaneously

A checkbox is a boolean, so an empty string will leave the box unchecked, and anything (including a whitespace) will check the box. If you want the user to be able to check multiple boxes at once and grab their individual values, they each need their own data variable. uncheckMePlease: 'make this truthy', If the user checks one box, all other boxes that are bound to this variable will also be checked.
